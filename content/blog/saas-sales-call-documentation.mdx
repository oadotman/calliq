# Sales Call Documentation for SaaS Companies: The Complete Guide

SaaS sales calls are complex. Multiple stakeholders, technical requirements, integration discussions, and pricing negotiations all happen across dozens of touchpoints. Here's how to document them properly without losing your mind.

- Single decision maker

- Linear sales process

**SaaS Sales Reality:**

- Complex technical requirements
- Multiple evaluation stages
- Security reviews
- Integration discussions
- Pilot programs
- Renewal considerations

## The SaaS Call Documentation Framework

### 1. Discovery Call Documentation

**Must Capture:**
- Current tech stack
- Integration requirements
- Team size and structure
- Budget and timeline
- Decision process
- Success criteria
- Pain points ranking
- Champion identification

**Example Template:**
```
DISCOVERY CALL: [Company Name]
Date: [Date]
Attendees: [Names and Roles]

CURRENT STATE:
- Tech Stack: [List all mentioned tools]
- Team Size: [Number and structure]
- Current Process: [How they work today]
- Pain Points: [Ranked 1-5]

DESIRED STATE:
- Goals: [What success looks like]
- Timeline: [When they need solution]
- Budget: [Range or constraints]

TECHNICAL:
- Integrations Needed: [List all]
- Security Requirements: [Compliance needs]
- Volume: [Users/data/transactions]

NEXT STEPS:
- Technical demo on [date]
- Security review needed: [Y/N]
- Stakeholders to involve: [Names]
```

### 2. Demo Call Documentation

**Critical Elements:**
- Features demonstrated
- Questions asked
- Objections raised
- Excitement indicators
- Confusion points
- Custom requirements
- Competition mentioned

**The Demo Reaction Map:**
```
FEATURE | REACTION | QUESTIONS | CONCERNS
--------|----------|-----------|----------
Feature A | Excited | How customizable? | None
Feature B | Neutral | API available? | Complexity
Feature C | Concerned | None | Too manual
```

### 3. Technical Evaluation Documentation

**Track Everything:**
- API questions
- Security concerns
- Integration timelines
- Data migration needs
- Performance requirements
- Compliance needs
- Custom development requests

### 4. Pricing and Negotiation Documentation

**Document All Variables:**
- Initial quote provided
- Discount discussions
- Payment terms
- Contract length
- User/usage tiers
- Add-on services
- Success metrics

## Documentation Best Practices for SaaS

### The MEDDIC Documentation Method

**Metrics:**
- Quantified pain points
- ROI calculations
- Success metrics

**Economic Buyer:**
- Name and role
- Budget authority
- Concerns

**Decision Criteria:**
- Evaluation factors
- Scoring methodology
- Comparison framework

**Decision Process:**
- Steps and timeline
- Stakeholders involved
- Approval requirements

**Identify Pain:**
- Ranked problems
- Impact assessment
- Urgency level

**Champion:**
- Internal advocate
- Influence level
- Commitment indicators

### The Handoff Documentation

**From Sales to Customer Success:**
- Use case details
- Implementation timeline
- Success criteria
- Key stakeholders
- Technical requirements
- Promises made
- Risk factors

**The Handoff Template:**
```
CUSTOMER HANDOFF DOCUMENT
Account: [Company]
ACV: [Amount]
Close Date: [Date]

KEY INFORMATION:
- Primary Use Case: [Specific workflow]
- Success Metrics: [What they measure]
- Go-Live Date: [Target date]
- Champions: [Names and roles]

TECHNICAL DETAILS:
- Integrations: [List with priority]
- Data Migration: [Scope and timeline]
- Custom Requirements: [Any promises]

RISK FACTORS:
- Competition: [If evaluating others]
- Concerns: [Unresolved objections]
- Dependencies: [External factors]

COMMITMENTS MADE:
- [List any specific promises]
- [Include who made them]
- [Note timeline committed]
```
